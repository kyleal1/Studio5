Sprint 4


What was done?

This was by far one of our best sprints in terms of productivity and being agile.

We were asked to create profiles, show what games are popular, add genres, sort our data, add images and add a feature for disabling accounts.

During this sprint, I was asked to add genres, and add a popular games column which showed which games were the most popular.

For me, this was a lot more technically challenging as I hadn't done a full stack developement yet, and had lots to learn but was confident in taking on the task.

What I did

During this sprint, I took on the task of adding genres and adding a popular games column on our home page.

Adding the genres was a full stack developement, where I created a model, controller, migration and database seeder class. This was by far the most challenging task I had taken on, but I was confident as I had learnt a lot along the way, was willing to learn more and wanted something technically challenging.

I had to create a model, migration and table. The model had the relationship that a genre can have many games and a game can have one genre. I then had to add a foreign key constraint, seed the database, edit the views and controllers, and then test the new feature.

The other task I took on was adding popular games column. This meant I had to edit the view to add a column, add a query in the controller and then reference that in the view.

The popular games query was of the count of individual scores added. The game with the most was the most popular. This was then sorted in descending order, so the most popular game was at the top.

Personal achievements

During this Sprint, I felt I took on a much larger and difficult workload than before. This meant I was going to need to focus more time and effort onto this sprint as to not let the team or myself down

I had to watch tutorials and read websites to get a good grasp of what I was dealing with.

At one point, I got stuck on how to create the query for the popular games. I went to our scrum master, Irving, who had difficulty too. We both researched and came up with a solution and appropriate query which achieved exactly what we wanted to achieve.
